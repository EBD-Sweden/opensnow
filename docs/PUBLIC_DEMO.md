# OpenSnow public demo

This is the safest one-command path for external testers who have a fresh clone and Rust installed. It uses only local loopback services, generated sample rows, and repo-local state under `.opensnow-demo/`.

## One command

```bash
scripts/demo.sh
```

What the command does:

1. Writes `.opensnow-demo/opensnow.toml` with local-only paths and ports.
2. Starts `cargo run -p opensnow-cli -- --config .opensnow-demo/opensnow.toml start` if `http://localhost:8080/health` is not already healthy.
3. Loads the deterministic manifest at `demo/public-demo-manifest.json` through `scripts/demo-seed.py`.
4. Runs REST query checks from the manifest.
5. Runs `scripts/public-smoke.sh` with `OPENSNOW_SKIP_PG=1` by default so `psql` is optional for first-time testers.
6. Prints the Web UI, health URL, manifest path, state directory, log path, and reset command.

## Stable demo data manifest

Manifest: `demo/public-demo-manifest.json`

Properties:

- `synthetic: true`
- `deterministic: true`
- `seed: 424242`
- `contains_real_customer_data: false`
- Source is generated, not copied from users, customers, vendors, logs, production systems, or private datasets.

Tables loaded by `scripts/demo-seed.py`:

- `demo_regions` — 4 generated region rows.
- `demo_customers` — 6 generated customer rows.
- `demo_orders` — 12 generated order rows.

Example query after `scripts/demo.sh`:

```bash
curl -fsS -H 'content-type: application/json' \
  -d '{"sql":"SELECT status, COUNT(*) AS orders FROM demo_orders GROUP BY status ORDER BY status"}' \
  http://localhost:8080/api/v1/query
```

## Cleanup and reset

```bash
scripts/demo.sh reset
```

The reset command stops the process recorded in `.opensnow-demo/opensnow.pid` and removes `.opensnow-demo/`. It refuses to delete a custom `OPENSNOW_DEMO_HOME` outside the repo-owned `.opensnow-demo/` path; remove custom paths manually if you intentionally override them.

## Optional pgwire smoke

The one-command demo skips pgwire by default because many fresh machines do not have `psql` installed and the current pgwire path is trusted-local only. Server enablement and client smoke are separate switches:

- `OPENSNOW_ENABLE_PGWIRE=1` makes `scripts/demo.sh` start OpenSnow with `--enable-pgwire`.
- `OPENSNOW_SKIP_PG=0` lets `scripts/public-smoke.sh` run `psql` and Python PostgreSQL-client checks.

To start trusted-local pgwire and include pgwire smoke once `psql` is available:

```bash
OPENSNOW_ENABLE_PGWIRE=1 OPENSNOW_SKIP_PG=0 scripts/demo.sh
```

## Safety boundary

- No cloud credentials are required.
- No auth tokens are generated by the demo command.
- No real customer data is bundled.
- Hosted or marketplace demos still require the enterprise-auth and security gates in `docs/ENTERPRISE_AUTH_QA_VALIDATION.md` and `docs/QA_RELEASE_CHECKLIST.md`.
