Copyable commands

pgwire / psql (trusted local only)
pgwire is disabled by default for public demos.
Enable only for trusted local smoke tests with --enable-pgwire, [server].pg_enabled=true, or OPENSNOW_ENABLE_PGWIRE=1.
psql -h localhost -p 5433
REST API query
curl http://localhost:8080/api/v1/query \
  -H 'content-type: application/json' \
  -d '{"sql":"SELECT 1 AS smoke;"}'
Load demo data API
curl -X POST http://localhost:8080/api/v1/demo/load

Clear error states: connection errors usually mean the OpenSnow server is not running or the wrong port is open. SQL errors include known limits and docs links where available.

Public demo boundary: pgwire is disabled by default for public demos. Enable it only for trusted local compatibility smoke tests after reading docs/PUBLIC_TEST_PATH.md; current pgwire support is simple-query only.

BI/dbt introspection: the supported compatibility pack is psql simple-query, basic information_schema, and REST /api/v1/dbt/catalog. Python drivers, COPY, extended query protocol, and general pg_catalog BI probes are not certified yet and should fail clearly.

Evaluation sandbox boundary: hosted sandbox accounts use /api/v1/evaluation/register, generated eval-* tenants, tracked quotas, and operator suspend/revoke controls. They are not enterprise BYOC/customer-owned infrastructure accounts.

Docs: docs/DEPLOYMENT.md, docs/SQL_COMPATIBILITY.md, docs/PUBLIC_TEST_PATH.md
